BACKEND API ROUTES (Express):

POST /api/roast
- Input: { tokenIdea, tokenName, ticker }
- Make 3 parallel Gemini calls for personas
- Make 1 Gemini call for scoring
- Return: { bull, skeptic, rugDetector, score }
- Save to history array (max 10)
- Never expose GEMINI_API_KEY to frontend

GET /api/trending  
- Fetch Four.meme trending tokens
- Try: https://four.meme/api/v1/token/list?sort=trending&limit=20
- Backup: https://api.four.meme/token/list
- If both fail: return mock trending data so app never breaks
- Run Gemini to classify dominant narrative
- Return: { narrative, tokens, metaBadge }

GET /api/history
- Return last 10 roasts from memory array

POST /api/history
- Save new roast to memory array
- Keep max 10, remove oldest

GET /api/votes/:roastId
- Return vote counts for a roast

POST /api/votes/:roastId
- Add a vote (bull/skeptic/rug)
- Return updated counts

POST /api/battle
- Input: { coinA, coinB }
- Run two parallel roasts
- Run Gemini to declare winner
- Return both roasts + winner declaration

GET /api/meta
- Return meta intelligence data
- Trending narratives, timing data, graveyard tokens
- Use Gemini to generate forecast
